# Yeka Cleaning Management System - Role Usage Documentation

## Overview

This document describes each user role's purpose, permissions, and typical usage within the Yeka Cleaning Management System.

---

## Role Definitions

### 1. Super Administrator (`superadmin`)

**Access Level:** 4 (Maximum)

**Usage Context:**
- Highest-level system administrator with complete control over all operations
- System configuration, user management, financial oversight, and policy enforcement
- Has read/write/delete access to every module

**Primary Responsibilities:**
- Manage all system users (create, update, deactivate)
- Configure system-wide settings and permissions
- View and generate all financial reports (receipts, payroll, capital)
- Oversee partnerships with outsource and private cleaning companies
- Manage company capital and transaction records
- Create and publish system announcements/posts
- Generate PDF reports of all system data
- Access all modules of all other roles (read-only for sensitive data)

**Daily Activities:**
- Review system activity logs and audit trails
- Approve new user registrations from other managers
- Monitor financial transactions and capital balance
- Generate and distribute daily/weekly management reports
- Update system policies and publish posts
- Resolve escalated issues from managers and staff
- Backup and system integrity checks

**Modules Access:**
- Dashboard → Home
- Users Management (CRUD)
- Checklist Management
- Register Receipt (Create/Edit/Delete)
- Outsource Company Management
- Private Cleaning Company Management
- Capital Transactions (View and Manage)
- Receipt & Payroll (Dual-tab interface)
- All PDF Reports Generation
- Post Management (News/Announcements)
- Role Usage & Permissions Management

**Financial Access:** Full - Can view all revenue, expenses, payroll, and capital

**Special Permissions:**
- Can create users for ANY role
- Can delete/modify any record
- Can export all data as PDF
- Can edit role definitions and permissions
- Can view system analytics

---

### 2. Manager (`manager`)

**Access Level:** 3 (High)

**Usage Context:**
- Operational manager overseeing a team of staff/cleaners
- Manages day-to-day operations, client relationships, and service quality
- Reports to Super Admin

**Primary Responsibilities:**
- Manage assigned staff members (schedule, performance)
- Review and approve/decline service bookings
- Monitor ongoing service delivery and quality
- Generate operational reports for their team
- Handle customer escalations and complaints
- Coordinate resource allocation (staff, equipment, vehicles)

**Daily Activities:**
- Review daily service schedule
- Assign tasks to staff members
- Check booking confirmations for the day
- Monitor services in progress via dashboard
- Address customer feedback/issues
- Prepare daily operational summary

**Modules Access:**
- My Staff Dashboard (view team performance)
- My Staff Detail Pages (CRUD for assigned staff)
- My Bookings (view and manage bookings)
- Services Management
- Reports (team-specific)

**Financial Access:** Limited - Can view revenue generated by their team only

**Reports Access:**
- Team performance metrics
- Booking conversion rates
- Service completion statistics
- Revenue per staff member

---

### 3. Staff Member (`staff`)

**Access Level:** 2 (Standard)

**Usage Context:**
- Core operational workforce
- Executes cleaning services, maintains records, handles administrative tasks
- Supports dispatch and coordination functions

**Primary Responsibilities:**
- Perform assigned cleaning services (residential, commercial, special)
- Complete daily checklists and service reports
- Maintain updated service records in system
- File agency reports and Yaka reports
- Manage office resources (library, equipment)
- Coordinate with dispatch and wereda officers

**Daily Activities:**
- View daily assigned tasks (via Dashboard/Service)
- Follow office plan for resource allocation
- Access library for procedure documents
- Complete checklist items for each job
- Submit agency reports weekly
- Upload Yaka reports as required
- Check dispatch schedules
- View outsource receipts and office recognition

**Modules Access:**
- Service → View tasks, update status
- Office Plan → View floor plans, office layouts
- Library → Access documents, manuals
- Agency Report → Create/view reports
- Checklist → Complete daily/weekly tasks
- Yaka Report → Submit periodic reports
- Subcity Officer → View contact list
- Subcity Driver → View assigned drivers
- Wereda Officer → View wereda contacts
- Dispatch Schedule → View upcoming dispatches
- Outsource Receipt → View receipts
- Office Recognition → View awards
- Gallery → View team photos
- Training → Access training materials
- Contact → Internal messaging
- Settings → Personal preferences

**Financial Access:** No direct financial access (view-only on receipts assigned to them)

---

### 4. Cleaner (`cleaner`)

**Access Level:** 1 (Basic)

**Usage Context:**
- Front-line cleaning personnel working at client sites
- Mobile workforce requiring minimal system complexity
- Focus on task completion and time logging

**Primary Responsibilities:**
- Perform residential and commercial cleaning services
- Follow standardized cleaning procedures (checklists)
- Maintain equipment and report issues
- Log hours worked and services completed
- Report completion status to dispatch

**Daily Activities:**
- View assigned jobs for the day
- Navigate to client location
- Start job timer upon arrival
- Complete service following checklist
- Mark job as complete in system
- Submit any issues/notes

**Modules Access:**
- Dashboard → Overview of assigned tasks
- My Jobs → List of today's assignments
- Schedule → View upcoming appointments

**Financial Access:** None (only view own completed jobs/earnings summary)

---

### 5. Registered User / Customer (`user`)

**Access Level:** 1 (Basic - External)

**Usage Context:**
- Individual or business clients who book cleaning services
- External-facing user with limited access to booking features
- Separate from internal staff roles

**Primary Responsibilities:**
- Browse available cleaning services
- Book appointments (date/time/service type)
- Manage booking history
- View and download receipts
- Update contact information
- Contact support

**Daily Activities:**
- View service catalog and pricing
- Book new cleaning appointments
- Check upcoming bookings
- View past service history
- Download payment receipts
- Contact office for inquiries

**Modules Access:**
- Dashboard → Personal dashboard
- Services → Browse and select
- Bookings → Create and manage appointments
- Profile → Personal details
- Gallery → View work samples
- Contact → Send messages

**Financial Access:** View own receipts only

---

### 6. Wereda Mahberat Officer (`wereda_mahberat`)

**Access Level:** 2 (Standard - Financial/Admin)

**Usage Context:**
- Government administrative officer at Wereda level
- Manages receipt collection, payroll records, and capital tracking for their jurisdiction
- Acts as liaison between Yeka and Wereda administration

**Primary Responsibilities:**
- Record and manage monthly receipts from clients
- Process payroll for local staff (approval and tracking)
- Maintain capital transaction ledger for Wereda
- Generate monthly financial reports
- Coordinate with Dispatch Officer for field operations
- Maintain Wereda-specific gallery and contact directory

**Daily Activities:**
- Enter daily receipts into system
- Review and approve payroll entries
- Update capital transaction records
- Check meeting room availability (if needed)
- Communicate with dispatch for logistics
- Generate monthly receipt summary

**Modules Access:**
- Dashboard → Financial overview
- Monthly Receipt → CRUD operations
- Payroll Report → View and manage payroll
- Capital → Transaction management
- Gallery → Wereda photos
- Contact → Contact management
- All Mahberat Report → Read-only access (via dispatch)

**Financial Access:** Full view of Wereda-level receipts, payroll, and capital

**Special Permissions:**
- Can view all monthly receipts
- Can process payroll entries
- Can create/update capital transactions
- Can generate financial reports for Wereda

---

### 7. Dispatch Officer (`dispatch_officer`)

**Access Level:** 3 (High - Operations)

**Usage Context:**
- Logistics and coordination hub
- Manages meeting facilities, dispatch creation, and field communications
- Central coordination point between offices and field teams

**Primary Responsibilities:**
- Schedule and allocate meeting rooms for meetings/events
- Create and track dispatch orders (materials, documents, equipment)
- Monitor dispatch status (pending, in-transit, delivered)
- Generate Mahberat reports for Wereda officers
- Maintain gallery of dispatch operations
- Handle incoming/outgoing communications

**Daily Activities:**
- Review dispatch pending list
- Create new dispatch orders with details
- Assign drivers to dispatches
- Track dispatch progress (check-in/check-out)
- Reserve meeting rooms for meetings
- Generate Mahberat reports as requested
- Contact drivers and wereda officers

**Modules Access:**
- Dashboard → Dispatch overview
- Meeting Room → CRUD for rooms, reservations
- All Mahberat Report → Generate and distribute reports
- Create Dispatch → Create/edit/delete dispatches
- Gallery → Operational photos
- Contact → Driver/officer contact list

**Financial Access:** None (no direct financial transactions)

**Special Permissions:**
- Full dispatch management (create, assign, track)
- Meeting room booking control
- Report generation for Wereda consumption
- Direct communication with drivers via contact module

---

### 8. Driver (`driver`)

**Access Level:** 1 (Basic - Field Operations)

**Usage Context:**
- Vehicle operator executing dispatch orders
- Mobile workforce requiring simple, clear task list and status updates
- Minimal system interaction while driving

**Primary Responsibilities:**
- Follow dispatch instructions exactly
- Complete delivery/pickup tasks on time
- Update task status (en route, arrived, completed)
- Report GPS location periodically
- Communicate issues to dispatch
- Maintain vehicle condition logs

**Daily Activities:**
- View list of assigned dispatches
- Check pickup/dropoff details and addresses
- Start task timer when beginning journey
- Mark task as picked up/delivered
- Update location (manual or auto-GPS)
- View contact info for support

**Modules Access:**
- Dashboard → Today's tasks summary
- Location → View/Update current location
- Task → View assignments, update status
- Contact → Emergency contacts, support

**Financial Access:** None

**Special Permissions:**
- Can update own task status only
- Can update own location
- Cannot view other drivers' tasks
- Cannot create/modify dispatches (read-only assigned tasks)

## Role Usage Examples

### Example 1: Monthly Workflow for Each Role

**Super Admin:**
- Week 1: Review new user registrations, approve manager requests
- Week 2: Generate quarterly financial PDF report
- Week 3: Update role permissions if needed
- Week 4: Publish monthly newsletter post

**Manager:**
- Daily: Review staff assignments, approve urgent bookings
- Weekly: Generate team performance report
- Monthly: Submit team revenue summary to Super Admin

**Staff:**
- Daily: Complete assigned checklists, submit agency reports
- Weekly: Attend training, update Yaka report
- Monthly: Participate in office recognition nominations

**Wereda Mahberat:**
- Daily: Enter daily receipts, reconcile balances
- Weekly: Review pending payroll, approve entries
- Monthly: Generate monthly financial report for Wereda, submit to Super Admin

**Dispatch Officer:**
- Daily: Create dispatches for morning deliveries, track afternoon returns
- Weekly: Reserve meeting rooms for upcoming meetings
- Monthly: Generate All Mahberat Report for Wereda offices

**Driver:**
- Daily: Check morning dispatch assignments, complete deliveries, update status
- Weekly: Check vehicle maintenance status
- Monthly: Submit driver activity summary

### Example 2: Data Flow by Role

**Receipt Flow:**
1. **User** receives service → pays → gets receipt from **Staff**
2. **Staff** enters receipt → saved to receipts table
3. **Wereda Mahberat** aggregates monthly receipts → generates MonthlyReceipt record
4. **Super Admin** reviews all receipts → exports PDF financial report

**Dispatch Flow:**
1. **Staff** requests dispatch → creates entry
2. **Dispatch Officer** reviews → creates official Dispatch record
3. **Driver** receives notification → completes task → updates status
4. **Wereda Mahberat** views All Mahberat Report generated by Dispatch Officer

## Database Tables Reference

### Role-Related Tables

**role_definitions** - Detailed role metadata (description, permissions, usage)
- Used by: SuperAdmin (RoleUsage page)
- Populated by: DatabaseSeeder

**role_permissions** - Module-level permissions per role
- Defines CRUD access per module per role
- Used for access control logic (future enhancement)

**role_activity_logs** - Audit trail of user actions
- Records: who did what, when, where
- Used for security auditing and usage analytics

**system_usage_analytics** - Aggregated usage metrics
- Tracks activity by role (logins, page views, actions)
- Used for performance monitoring

## Setting Up New Users

### For Super Admin (Manual Setup):
1. Navigate to `/Dashboard/SuperAdmin/Users`
2. Click "Add User"
3. Fill in name, email, password, select role
4. User will be redirected to appropriate dashboard on login

### For Other Roles (Manager/Self-registration):
- Currently all user creation is via Super Admin only
- Future: Add registration form with role-based approval workflow

## Best Practices

### For Super Admins:
- Regularly review role_activity_logs for unusual activity
- Update role_definitions when introducing new modules
- Backup database before permission changes
- Keep user roles minimal - grant only necessary access

### For Managers:
- Delegate tasks based on staff capacity and training
- Review checklist completions daily
- Escalate system issues to Super Admin

### For Staff:
- Complete checklists for every job
- Submit reports on time
- Keep contact information updated

### For Wereda Mahberat:
- Reconcile receipts daily to avoid backlog
- Verify payroll entries before approval
- Maintain audit trail for financial transactions

### For Dispatch Officers:
- Confirm driver availability before assigning
- Update dispatch status immediately on change
- Archive completed dispatches monthly

### For Drivers:
- Update location when on the move
- Mark tasks as complete immediately after delivery
- Report vehicle issues promptly

## Support & Escalation

**Role Hierarchy Support Chain:**

```
Super Admin (Level 4)
    ↑
Manager (Level 3) - for Staff issues
    ↑
Wereda Mahberat (Level 2) - for financial/receipt issues
    ↑
Dispatch Officer (Level 3) - for dispatch/logistics issues
    ↑
Driver/Staff (Level 1) - field operations
```

**Contact Channels:**
- Technical Issues: Super Admin
- Operational Issues: Manager or Wereda Mahberat
- Dispatch Issues: Dispatch Officer
- Field Issues: Driver → Dispatch → Wereda
